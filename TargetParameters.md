# Полный справочник по параметрам конфигурации AM32 (targets.h)

В этом документе подробно описаны **все** директивы `#define`, которые можно использовать при создании собственной конфигурации платы в файле `targets.h` прошивки AM32. Параметры сгруппированы по функциональному назначению. Для каждого параметра указан тип значения, значение по умолчанию (если есть) и пояснение.

> **Важно:** Если какой-либо параметр не задан в секции вашей платы, будет использовано значение по умолчанию из нижней части `targets.h`.

## Идентификация прошивки

### `FIRMWARE_NAME`
- **Тип:** строка
- **Назначение:** Имя прошивки, которое отображается в конфигураторе AM32 и в данных телеметрии. Допускается длина до 12 символов (часто дополняется пробелами).
- **Пример:** `"Neutron AIO"`

### `FILE_NAME`
- **Тип:** строка
- **Назначение:** Имя файла для выходного бинарного файла прошивки при сборке. Используется в скриптах сборки для именования `.bin` или `.hex`.
- **Пример:** `"NEUTRON_2_6S_AIO_F421"`

## Параметры выходного каскада

### `DEAD_TIME`
- **Тип:** целое число
- **Назначение:** «Мёртвое время» (dead time) между выключением верхнего и включением нижнего плеча полумоста, в наносекундах (нс). Зависит от используемого MOSFET-драйвера и силовых транзисторов. Слишком маленькое значение может привести к сквозным токам, слишком большое — к увеличенным потерям.
- **Пример:** `40` (40 нс)

### `USE_INVERTED_HIGH`
- **Тип:** флаг (определён / не определён)
- **Назначение:** Если определён, верхние плечи управляются инвертированным сигналом (активный низкий уровень). Используется для некоторых драйверов с инвертированными входами.
- **Пример:** `#define USE_INVERTED_HIGH`

### `USE_INVERTED_LOW`
- **Тип:** флаг
- **Назначение:** Инвертирует управление нижними плечами (активный низкий уровень).
- **Пример:** `#define USE_INVERTED_LOW`

### `USE_OPEN_DRAIN_LOW` / `USE_OPEN_DRAIN_HIGH`
- **Тип:** флаг
- **Назначение:** Переводят соответствующие выходы в режим открытого стока (open drain). Применяется в специфических схемотехнических решениях.

## Группа аппаратного обеспечения

Параметр, определяющий распиновку микроконтроллера — пины ШИМ, компараторов, входной сигнал и каналы DMA. Необходимо выбрать **один** из вариантов, соответствующих вашей плате.

### `HARDWARE_GROUP_<СЕМЕЙСТВО>_<БУКВА>`
- **Тип:** макроопределение
- **Назначение:** Группа пинов и периферии.
- **Примеры:**
  - `HARDWARE_GROUP_AT_B` — для AT32F421 вариант B
  - `HARDWARE_GROUP_AT_C` — для AT32F421 вариант C
  - `HARDWARE_GROUP_F0_A`, `F0_B`, … — для STM32F051
  - `HARDWARE_GROUP_G0_N` — для STM32G071 (NeutronRC)
  - `HARDWARE_GROUP_L4_B` — для STM32L431 вариант B
  - и т.д.

### Группы порядка фаз (компараторов)

Дополняют `HARDWARE_GROUP` и задают, какой пин компаратора соответствует фазе A, B или C.

- **Тип:** макроопределение (выбрать одно)
- **Назначение:** Устанавливает соответствие «пин компаратора → фаза мотора».
- **Примеры:**
  - `HARDWARE_GROUP_AT_045` → фаза A: PA0, B: PA4, C: PA5
  - `HARDWARE_GROUP_AT_540` → A: PA5, B: PA4, C: PA0
  - `HARDWARE_GROUP_AT_450` → A: PA4, B: PA5, C: PA0
  - `HARDWARE_GROUP_AT_054` → A: PA0, B: PA5, C: PA4
  - `HARDWARE_GROUP_AT_405` → A: PA4, B: PA0, C: PA5
  - `HARDWARE_GROUP_AT_245` → A: PA2, B: PA4, C: PA5 (специальный для polling mode)
  - Для STM32F0: `HARDWARE_GROUP_F0_045`, `F0_450`, `F0_540` и т.д.
  - Для L431: `HARDWARE_GROUP_L4_045`, `L4_054`, `COMP_ORDER_L4_A_045`, `COMP_ORDER_L4_A_540`

## Измерение тока

### `MILLIVOLT_PER_AMP`
- **Тип:** целое число (мВ/А)
- **По умолчанию:** `20`
- **Назначение:** Коэффициент перевода напряжения на входе АЦП в измеряемый ток. Указывает, сколько милливольт соответствует току в 1 Ампер на конкретной плате (с учётом шунта и усилителя). Вычисляется по формуле: `(Напряжение на АЦП в мВ) / Ток (А)`.  
  **Значение должно соответствовать фактической схеме.**
- **Пример:** `80` (для датчика 10 мВ/А с усилением, дающим 80 мВ/А на АЦП)

### `CURRENT_OFFSET`
- **Тип:** целое число
- **По умолчанию:** `0`
- **Назначение:** Смещение нуля тока в милливольтах. Напряжение на входе АЦП при нулевом токе. Используется для компенсации постоянной составляющей.
- **Пример:** `498` (498 мВ)

### `CURRENT_ADC_PIN`
- **Тип:** макроопределение пина
- **Назначение:** Пин микроконтроллера, к которому подключён датчик тока.
- **Пример:** `GPIO_PINS_2` (для AT32), `LL_GPIO_PIN_5` (для STM32)

### `CURRENT_ADC_CHANNEL`
- **Тип:** макроопределение канала АЦП
- **Назначение:** Номер канала АЦП, соответствующий выбранному пину.
- **Пример:** `ADC_CHANNEL_2` (AT32), `LL_ADC_CHANNEL_5` (STM32)

Если эти два параметра не заданы, используются значения по умолчанию, определённые в секции `MCU_...` ниже.

## Измерение напряжения

### `TARGET_VOLTAGE_DIVIDER`
- **Тип:** целое число
- **По умолчанию:** `110`
- **Назначение:** Коэффициент делителя напряжения батареи. Рассчитывается как `(R1 + R2) / R2`, где R1 — верхнее плечо (к плюсу батареи), R2 — нижнее (к земле). Используется для пересчёта измеренного напряжения на пине АЦП в реальное напряжение батареи.
- **Пример:** `210` для делителя 1:21 (например, 200k и 10k → (200+10)/10 = 21 → умножаем на 10? В коде множитель 210 соответствует `21 * 10`).

### `VOLTAGE_ADC_PIN`
- **Тип:** макроопределение пина
- **Назначение:** Пин, измеряющий напряжение батареи (после делителя).
- **Пример:** `GPIO_PINS_6`

### `VOLTAGE_ADC_CHANNEL`
- **Тип:** канал АЦП
- **Назначение:** Номер канала для измерения напряжения.
- **Пример:** `ADC_CHANNEL_6`

> **Примечание:** Для некоторых МК также используются устаревшие названия `CURRENT_SENSE_ADC_PIN`, `VOLTAGE_SENSE_ADC_PIN`.

## Температурный датчик NTC

### `USE_NTC`
- **Тип:** флаг
- **Назначение:** Включает измерение температуры с помощью NTC-термистора.
- **Пример:** `#define USE_NTC`

### `NTC_ADC_PIN`
- **Тип:** пин
- **Назначение:** Пин, к которому подключён термистор.
- **Пример:** `GPIO_PINS_3`

### `NTC_ADC_CHANNEL`
- **Тип:** канал АЦП
- **Назначение:** Канал АЦП для термистора.
- **Пример:** `ADC_CHANNEL_3`

### `USE_ADC_1_2`
- **Тип:** флаг
- **Назначение:** Использовать два АЦП (если требуется для дополнительных измерений, например, на STM32G431). Указывает на то, что задействованы и ADC1, и ADC2.

### `PA6_NTC_ONLY`
- **Тип:** флаг
- **Назначение:** Включает режим, при котором пин PA6 используется только для NTC (обычно на этом пине может висеть и телеметрия).

## Телеметрия и коммуникация

### `USE_SERIAL_TELEMETRY`
- **Тип:** флаг
- **Назначение:** Включает поддержку ESC-телеметрии по UART (обычно с использованием одного провода). Если не определён, телеметрия будет отключена.
- **Пример:** `#define USE_SERIAL_TELEMETRY`

### `USE_PA14_TELEMETRY`
- **Тип:** флаг
- **Назначение:** Использовать вывод PA14 в качестве пина телеметрии (альтернативное размещение). Актуально для некоторых плат.
- **Пример:** `#define USE_PA14_TELEMETRY`

### `NO_PA11_PA12_REMAP`
- **Тип:** флаг
- **Назначение:** Запрещает переназначение пинов PA11/PA12 (обычно используется для USB). Применяется, если эти пины задействованы для других целей.

## Светодиодная индикация (LED Strip)

### `USE_LED_STRIP`
- **Тип:** флаг
- **Назначение:** Включает управление адресной светодиодной лентой (WS2812B или аналоги).
- **Пример:** `#define USE_LED_STRIP`

### `WS2812_PIN`
- **Тип:** пин
- **Назначение:** Пин микроконтроллера, к которому подключена лента.
- **Пример:** `GPIO_PINS_8`

### `USE_RGB_LED`
- **Тип:** флаг
- **Назначение:** Включает управление обычным RGB-светодиодом (не адресным) через отдельные пины для каждого цвета.

### `RED_PORT`, `RED_PIN`
- **Тип:** порт и пин
- **Назначение:** Порт и пин для красного канала RGB-светодиода.
- **Пример:** `GPIOB`, `GPIO_PIN_3`

### `GREEN_PORT`, `GREEN_PIN`
- **Тип:** порт и пин
- **Назначение:** Для зелёного канала.

### `BLUE_PORT`, `BLUE_PIN`
- **Тип:** порт и пин
- **Назначение:** Для синего канала.

### `USE_CUSTOM_LED`
- **Тип:** флаг
- **Назначение:** Включает кастомную логику индикации светодиодами (детали зависят от реализации).

### `USE_PULSE_OUT`

- **Назначение:** Включает генерацию импульсов, пропорциональных оборотам двигателя, на отдельном выводе.
- **Пример:** `#define USE_PULSE_OUT`

### `RPM_PULSE_PORT`

### `RPM_PULSE_PIN`

- **Пример:** `LL_GPIO_PIN_10`

## Настройки запуска и защиты

### `USE_STARTUP_BOOST`
- **Тип:** флаг
- **Назначение:** Включает форсированный запуск (увеличенный ток на старте) для надёжного раскручивания мотора.
- **Пример:** `#define USE_STARTUP_BOOST`

### `TARGET_STALL_PROTECTION_INTERVAL`
- **Тип:** целое число (микросекунды?)
- **По умолчанию:** `6500`
- **Назначение:** Временной интервал, используемый для детектирования остановки (заклинивания) мотора. Если в течение этого времени не зафиксирована коммутация, считается, что мотор застопорен, и ESC отключается.
- **Пример:** `9000`

### `TARGET_MIN_BEMF_COUNTS`
- **Тип:** целое число
- **По умолчанию:** зависит от МК (обычно 2–3)
- **Назначение:** Минимальное количество успешных детектирований BEMF (противо-ЭДС), необходимых для подтверждения работы мотора. Используется в алгоритме определения старта.
- **Пример:** `3`

### `NO_POLLING_START`
- **Тип:** флаг
- **Назначение:** Отключает фазу «polling» при старте мотора (индуктивное определение положения). Используется для плат, где polling нестабилен.
- **Пример:** `#define NO_POLLING_START`

### `POLLING_MODE_THRESHOLD`
- **Тип:** целое число
- **По умолчанию:** `2000`
- **Назначение:** Пороговое значение (в тиках таймера), выше которого ESC переходит в режим коммутации по BEMF вместо polling.

## Скорость нарастания оборотов (ramp)

### `RAMP_SPEED_LOW_RPM`
- **Тип:** целое число
- **По умолчанию:** `6`
- **Назначение:** Коэффициент скорости нарастания оборотов на низких оборотах (при коммутационном интервале > 250 мкс). Меньше число = более медленный разгон.
- **Пример:** `1` (очень медленный разгон)

### `RAMP_SPEED_HIGH_RPM`
- **Тип:** целое число
- **По умолчанию:** `16`
- **Назначение:** Коэффициент нарастания на высоких оборотах (интервал < 250 мкс).
- **Пример:** `1`

### `RAMP_SPEED_STARTUP`
- **Тип:** целое число
- **По умолчанию:** `2`
- **Назначение:** Коэффициент разгона во время старта мотора.

### `CUSTOM_RAMP`
- **Тип:** флаг
- **Назначение:** Включает использование кастомного алгоритма разгона (обычно вместе с `LOOP_FREQUENCY_HZ`).

### `VOLTAGE_BASED_RAMP`
- **Тип:** флаг
- **Назначение:** Разгон зависит от напряжения питания (более плавный старт при высоком вольтаже).

---

## Ограничение напряжения питания

### `ONE_TWO_CELL_MAX`
- **Тип:** флаг
- **Назначение:** Ограничивает максимальное напряжение питания до 2S (примерно 8.4 В). Используется для ESC, рассчитанных только на 1–2 банки.
- **Пример:** `#define ONE_TWO_CELL_MAX`

### `THREE_CELL_MAX`
- **Тип:** флаг
- **Назначение:** Ограничение до 3S (12.6 В).

---

## Частота основного цикла (Loop Frequency)

### `LOOP_FREQUENCY_HZ`
- **Тип:** целое число (Гц)
- **По умолчанию:** `20000` (20 кГц)
- **Назначение:** Частота основного управляющего цикла. Допускается изменение для специальных применений (например, 10000 для медленных ESC, 20000 — стандарт, до 24000 и выше при шасси 48 МГц).
- **Пример:** `500`

## Дополнительные выводы и режимы

### `USE_PA6_TEMP`
- **Тип:** флаг
- **Назначение:** Использовать PA6 для измерения температуры (обычно комбинировано с телеметрией, когда нет отдельного пина NTC).

### `USE_ADC_INPUT` (глобальный)
- **Комментарий в файле:** не используется в конкретных конфигурациях, закомментирован. Вероятно, устаревший флаг.

### `PA6_VOLTAGE`
- **Тип:** флаг
- **Назначение:** Использовать PA6 для измерения напряжения (вместо стандартного пина). Применяется на некоторых платах.

### `FIXED_DUTY_MODE`
- **Тип:** флаг
- **Назначение:** Режим фиксированной скважности (для отладки или специфичных ESC, не принимает сигнал управления).
- **Пример:** `#define FIXED_DUTY_MODE`

### `FIXED_DUTY_MODE_POWER`
- **Тип:** число
- **Назначение:** Задаёт фиксированную мощность в процентах (0–100), используемую в режиме фиксированной скважности.
- **Пример:** `100`

### `USE_DRV8328_NSLEEP` / `USE_DRV8328_NFAULT`
- **Тип:** флаг + пины
- **Назначение:** Поддержка драйвера DRV8328: управление режимом сна (NSLEEP) и чтение ошибки (NFAULT). Требуют определения `NSLEEP_PORT`, `NSLEEP_PIN`, `NFAULT_PORT`, `NFAULT_PIN`.
- **Пример:** `#define USE_DRV8328_NSLEEP`

## Настройки памяти EEPROM

### `EEPROM_START_ADD`
- **Тип:** адрес во flash (uint32_t)
- **По умолчанию:** зависит от МК и объёма памяти (например, `0x08007C00` для 32k, `0x0800F800` для 64k, `0x0801F800` для 128k).
- **Назначение:** Адрес во flash-памяти, с которого начинается эмуляция EEPROM для хранения настроек. **Менять не рекомендуется**, если не требуется особая раскладка памяти.

## DroneCAN

### `DRONECAN_SUPPORT`
- **Тип:** флаг (0/1)
- **По умолчанию:** `0`
- **Назначение:** Если установлен в `1`, то собирается версия прошивки с поддержкой протокола DroneCAN. Автоматически переопределяет размер памяти EEPROM на 128k раскладку.
- **Пример:** `#define DRONECAN_SUPPORT 1`

### `DRONECAN_NODE_NAME`
- **Тип:** строка
- **Назначение:** Уникальное имя узла DroneCAN.
- **Пример:** `"com.vimdrones.esc_dev"`

### `CAN_TERM_PIN`
- **Тип:** пин (в формате `GPIO_PORT_PIN(port, pin)`)
- **Назначение:** Пин управления CAN-терминатором (если есть).
- **Пример:** `GPIO_PORT_PIN(1, 3)` для PB3

### `CAN_TERM_POLARITY`
- **Тип:** 0 или 1
- **Назначение:** Полярность сигнала включения терминатора (0 – активный низкий, 1 – активный высокий).

---

## Внутренний усилитель тока

### `USE_INTERNAL_AMP`
- **Тип:** флаг
- **Назначение:** Указывает, что используется внутренний операционный усилитель микроконтроллера (например, в STM32L431). Влияет на конфигурацию АЦП.
- **Пример:** `#define USE_INTERNAL_AMP`

## Частота ШИМ (PWM)

### `NOMINAL_PWM`
- **Тип:** целое число (Гц)
- **По умолчанию:** `24000`
- **Назначение:** Задаёт номинальную частоту ШИМ (и коммутации). На основе этой частоты вычисляется значение автоперезагрузки таймера TIM1 (`TIM1_AUTORELOAD`). Может быть изменено для специфических требований (например, 48 кГц).
- **Пример:** `24000`

### `TIM1_AUTORELOAD`
- **Тип:** вычисляемое значение
- **По умолчанию:** рассчитывается автоматически как `(CPU_FREQUENCY_MHZ * 1000000 / NOMINAL_PWM) - 1`.
- **Назначение:** Прямое указание периода ШИМ-таймера. Обычно не задаётся вручную, но можно переопределить, если нужно точное значение.

## Как создать свою конфигурацию

1. Выберите уникальный идентификатор цели, например `#ifdef MY_BOARD_F421`.
2. Задайте обязательные идентификаторы: `FIRMWARE_NAME`, `FILE_NAME`, `DEAD_TIME`.
3. Укажите аппаратную группу (`HARDWARE_GROUP_...`).
4. Определите порядок фаз компараторов (`HARDWARE_GROUP_..._045`, `_540` и т.п.).
5. Добавьте `USE_SERIAL_TELEMETRY`, если нужна телеметрия.
6. Задайте пины и каналы для тока и напряжения (если отличаются от дефолтных).
7. Вычислите и пропишите `MILLIVOLT_PER_AMP`, `CURRENT_OFFSET`, `TARGET_VOLTAGE_DIVIDER` согласно схеме вашей платы.
8. При необходимости добавьте дополнительные опции: NTC, LED, DroneCAN и т.д.
9. Сохраните файл и выполните сборку с указанием вашей цели в скрипте сборки (например, `make MY_BOARD`).

Все определенные в секции параметры перекроют значения по умолчанию. Если какой-то параметр не указан — будет использовано значение из секции «по умолчанию», что в большинстве случаев безопасно, но может привести к неверным измерениям тока/напряжения.
