Это возможно, для этого нужно будет вручную создать и прошить HEX-файл с вашими настройками по правильному адресу. Логика довольно простая, но требует некоторой ручной работы.

Вот как это делается:

1. Находим адрес EEPROM: Для микроконтроллера AT32F421 вашей платы NEUTRON_2_6S_AIO_F421 конфигурация EEPROM хранится во Flash-памяти по адресу 0x08007C00 .
2. Готовим файл с настройками: Вам нужно записать "сырые" байты настроек в HEX-файл. Обычно прошивка AM32 хранит настройки в виде непрерывного блока данных. Если допустить, что структура конфигурации занимает 256 байт (это можно уточнить в исходниках), то для создания такого файла можно использовать, например, утилиту srec_cat.
3. "Собираем" прошивку: Теперь у вас есть два HEX-файла: основной файл прошивки (NEUTRON_2_6S_AIO_F421.hex) и ваш новый файл с настройками EEPROM. Их можно объединить в один командой:
   ```bash
   srec_cat firmware.hex -intel eeprom.hex -intel -o combined.hex -intel
   ```
   Или же прошить их один за другим с помощью ST-Link . В итоге у вас получится один HEX-файл, который можно загрузить в ESC с помощью STM32CubeProgrammer или командой make flash. После этого ESC сразу запустится с вашими настройками, без необходимости настраивать его через конфигуратор.

Хотя в интернете упоминается, что это возможно , точная структура EEPROM для вашей версии AM32 обычно описана в документации к проекту. Проще всего будет один раз настроить регулятор через конфигуратор, сразу после прошивки, а затем считать с него дамп EEPROM — это и будет ваш шаблон для будущих прошивок.
