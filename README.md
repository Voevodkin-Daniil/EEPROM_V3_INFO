Структура EEPROM для прошивки AM32 (Electronic Speed Controller) представляет собой объединение (`union`), которое отображает область памяти размером 192 байта на набор параметров конфигурации регулятора. Эти параметры хранятся во flash-памяти микроконтроллера (эмуляция EEPROM) и загружаются при старте. Доступ к ним осуществляется через глобальную переменную `eepromBuffer` типа `EEprom_t`. Ниже приведено подробное описание каждого поля.

## Общая информация
- Размер структуры – 192 байта.
- Поля, помеченные как `reserved`, зарезервированы для будущего использования или выравнивания.
- Версионирование (`eeprom_version` и `version`) позволяет контролировать совместимость при обновлении прошивки.
- Для чтения/записи всей структуры используются функции `read_flash_bin()` и `save_flash_nolib()`, которые работают с областью flash, выделенной под пользовательские данные.

## Описание полей

### Служебные поля
| Поле | Адрес | Описание |
|------|-------|----------|
| `reserved_0` | 0 | Зарезервировано |
| `eeprom_version` | 1 | Версия структуры EEPROM (используется для миграции настроек) |
| `reserved_1` | 2 | Зарезервировано |
| `version.major`<br>`version.minor` | 3–4 | Версия прошивки (major.minor) |

## Функции для работы с EEPROM
В коде объявлены функции:
- `read_flash_bin(uint8_t* data, uint32_t add, int out_buff_len)` – читает `out_buff_len` байт из flash по адресу `add` в буфер `data`.
- `save_flash_nolib(uint8_t* data, int length, uint32_t add)` – записывает `length` байт из буфера `data` во flash по адресу `add` (без использования библиотечных функций, вероятно, напрямую через регистры).

Обычно эти функции вызываются для сохранения текущей конфигурации после её изменения (например, через конфигуратор) и для загрузки при старте. Вся структура `EEprom_t` записывается/считывается целиком, либо отдельные поля могут обновляться по необходимости.

## Важные замечания
- **Эмуляция EEPROM** во flash требует стирания сектора перед записью, поэтому часто используется кэширование изменений и запись всего блока.
- При изменении структуры (добавлении новых полей) необходимо увеличивать `eeprom_version`, чтобы прошивка могла корректно обработать старые данные.
- Поля, помеченные как `reserved`, не должны использоваться, но их значения должны сохраняться при обновлении, чтобы не нарушить выравнивание.

## Заключение
Структура EEPROM в AM32 обеспечивает гибкую настройку ESC под различные двигатели, аккумуляторы и режимы работы. Правильное понимание её полей позволяет эффективно настраивать регулятор, а также вносить изменения в прошивку с сохранением обратной совместимости.
