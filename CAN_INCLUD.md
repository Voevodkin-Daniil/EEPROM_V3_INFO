```markdown
# Подключение DroneCAN к AM32 – модульное руководство

Этот документ описывает, как добавить поддержку шины CAN (DroneCAN) в вашу сборку прошивки AM32.  
Он предназначен для использования совместно с основным конфигурационным файлом `targets.h`.

---

## 0. Аппаратные требования

- **Микроконтроллер с аппаратным CAN-контроллером**  
  Например: STM32L431, STM32G071, STM32G431, AT32F415, GD32E230 (некоторые версии)  
  [ERROR] *AT32F421, STM32F051, STM32F031 не имеют CAN – DroneCAN на них невозможен.*

- **CAN-трансивер** (SN65HVD230, TJA1050 и т.п.)  
  Подключается к пинам `CAN_TX` и `CAN_RX` микроконтроллера.

- **Шина CAN**  
  Витая пара с терминаторами 120 Ом на обоих концах.

### Типовая схема подключения (со стороны ESC)

```
MCU CAN_TX  ---- TXD трансивера
MCU CAN_RX  ---- RXD трансивера
Трансивер CAN_H / CAN_L --- к линии CAN

Питание трансивера: обычно 3.3 В (зависит от модели)
```

Конкретные пины `CAN_TX` / `CAN_RX` заданы **аппаратной группой** (`HARDWARE_GROUP_*`) – смотрите её описание в `targets.h`.

---

## 1. Изменения в файле `targets.h`

Внутри секции вашей платы (или создайте новую) добавьте следующие директивы.

### Минимальный набор

```c
#ifdef MY_BOARD_CAN
    // ... остальные параметры (DEAD_TIME, HARDWARE_GROUP, телеметрия и т.д.)

    #define DRONECAN_SUPPORT 1
    #define DRONECAN_NODE_NAME "com.mycompany.esc_name"
#endif
```

### Расширенный набор – с управляемым терминатором и внешним кварцем

```c
#ifdef MY_BOARD_CAN_HSE
    #define DRONECAN_SUPPORT 1
    #define DRONECAN_NODE_NAME "com.example.esc"

    // Управляемый CAN-терминатор (если предусмотрен схемой)
    #define CAN_TERM_PIN GPIO_PORT_PIN(1, 3)   // PB3 (пример)
    #define CAN_TERM_POLARITY 1                // 1 = активный высокий, 0 = активный низкий

    // Внешний кварцевый резонатор (HSE) для точного тактирования CAN
    #define USE_HSE
    #undef HSE_VALUE
    #define HSE_VALUE 8000000           // частота кварца в Гц
    #define USE_HSE_BYPASS 0            // 0 = кварц, 1 = внешний генератор (bypass)
#endif
```

### Описание ключевых макросов

| Макрос                     | Тип               | Описание |
|----------------------------|-------------------|----------|
| `DRONECAN_SUPPORT`         | флаг (0 или 1)    | Включает стек DroneCAN. При `1` автоматически переводит раскладку EEPROM в режим 128 КБ flash (адрес `0x0801F800`). |
| `DRONECAN_NODE_NAME`       | строка            | Уникальное имя узла в сети CAN. Соглашение: `com.производитель.устройство` |
| `CAN_TERM_PIN`             | GPIO_PORT_PIN(..) | Пин, управляющий встроенным CAN-терминатором (необязательно). |
| `CAN_TERM_POLARITY`        | 0 или 1           | Полярность сигнала включения терминатора: `1` – высокий уровень включает, `0` – низкий. |
| `USE_HSE`                  | флаг              | Использовать внешний высокоскоростной генератор (кварц). |
| `HSE_VALUE`                | число (Гц)        | Частота внешнего кварца / генератора. |
| `USE_HSE_BYPASS`           | 0 или 1           | `0` – кварцевый резонатор; `1` – внешний тактовый сигнал (bypass mode). |

> !!! Если `USE_HSE` не определён, система будет тактироваться от внутреннего генератора (HSI). Для надёжного CAN **настоятельно рекомендуется** внешний кварц, особенно при скоростях 1 Мбит/с.

## 2. Существующие примеры конфигураций с CAN

В официальном `targets.h` можно найти следующие рабочие шаблоны:

- `TBS_12S_L431_CAN` – STM32L431, терминатор на PB3 (активный высокий), без HSE.
- `SEQURE_G431_CAN` – STM32G431, HSE 8 МГц, без управляемого терминатора.
- `VIMDRONES_NANO_L431_CAN` – HSE 24 МГц, без терминатора.
- `POGO_L431_CAN` – без терминатора и HSE.
- `TBS_F415_CAN` – AT32F415.

Используйте их как основу, копируя в свою секцию и заменяя имя платы.

## 3. Сборка прошивки

### 3.1. Makefile / скрипт сборки

Убедитесь, что ваша новая цель прописана в сборочном скрипте.  
Например, в `f421makefile.mk` (если вы используете AT32) или `Makefile`:

```make
TARGET ?= MY_BOARD_CAN
```

Или добавьте новую цель в список, если она не подхватывается автоматически.

### 3.2. Команды

```bash
make clean
make
```

После успешной компиляции в папке `Objects/` или `build/` появится файл `MY_BOARD_CAN.bin` (или `.hex`).

### 3.3. Прошивка

Загрузите полученную прошивку в ESC любым удобным способом:
- Через USB-UART адаптер и конфигуратор AM32.
- Через CAN-бутлоадер (если уже есть).
- Через SWD программатор.

---

## 4. Проверка работы в сети DroneCAN

После загрузки прошивки ESC должен появиться в сети CAN как узел с именем, заданным в `DRONECAN_NODE_NAME`.

Вы можете использовать:
- **Mission Planner** (вкладка DroneCAN)
- **UAVCAN GUI Tool**
- **Собственный CAN-сниффер**

Узел будет передавать стандартные сообщения ESC статуса (`uavcan.equipment.esc.Status`) и поддерживать динамическую конфигурацию через DroneCAN.

---

## 5. Важное замечание для AT32F421 (Neutron AIO и аналоги)

Микроконтроллер **AT32F421 не имеет аппаратного CAN-интерфейса**, поэтому:
- `DRONECAN_SUPPORT` **не будет работать** на данной платформе.
- Если вам нужен CAN, рассмотрите переход на другой ESC с MCU: STM32L431, G071, G431, AT32F415 и т.п.
- Альтернативно можно использовать внешний преобразователь UART‑CAN, но это потребует доработки кода прошивки.
